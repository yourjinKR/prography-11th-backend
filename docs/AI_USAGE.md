# AI 사용 내역

## 사용 목적
- 과제 문서 기반 구현 보조
- 코드 구조화 및 반복 작업 자동화
- 테스트 케이스 작성/보강
- 스펙 정합성 점검 항목 도출

## 사용 범위
- 초기 가이드 문서 작성 ([AGENT.md](../AGENT.md))
- 공통 응답/예외 처리 구조 생성
- 설계한 도메인과 스펙 간 정합성 확인
- 통합 테스트 작성 및 회귀 점검
- 제출 문서(README/ERD/아키텍처/정합성 기록) 작성

## 검증 방식
- 모든 변경은 로컬 테스트(`./gradlew test`)로 검증
- 스펙 문서와 응답 필드/에러코드/비즈니스 규칙을 수동 대조
- 발견된 불일치 항목은 [spec-alignment-issues.md](spec-alignment-issues.md)에 기록 후 수정

## 참고
- 최종 책임은 제출자에게 있으며, AI 생성 결과는 직접 검토/수정 후 반영
